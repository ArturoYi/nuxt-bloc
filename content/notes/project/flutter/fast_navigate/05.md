# navigationThrottleMs 导航防抖

> **阶段定位**：M1 后期的小而精工程化增强——解决生产环境「连点按钮重复 push」问题。  
> **代码位置**：`lib/src/internal/navigation_throttle.dart`  
> **接入点**：`FastRouterConfig` → `FastNavigator.bind` → 所有导航 API

---

## 一、问题从哪来

BarrelLock 联调时出现典型场景：

```
用户快速双击「进入详情」
  → FastNavigator.push('/detail') × 2
  → NavigationState 栈: [home, detail, detail]
  → 用户点返回，还在 detail，再点才回 home
```

这不是 bug，是 2.0「每次 push 都 faithfully 入栈」的正确行为。但**产品期望**是：短时间内重复导航意图应被丢弃。

go_router 没有内置此能力；1.0 时代常靠 `Debouncer` 或按钮 `AbsorbPointer` 在 UI 层 hack。fast_navigator 选择在**框架层统一拦截**。

---

## 二、心路历程：节流 vs 防抖 vs UI 层处理

| 方案 | 优点 | 缺点 | 结论 |
| --- | --- | --- | --- |
| UI 层 `AbsorbPointer` | 简单 | 每个按钮都要写，易漏 | ❌ |
| 全局 `Debouncer`（尾触发） | 只执行最后一次 | 导航应保留首次意图，不是最后一次 | ❌ |
| **Throttle（首触发 + 窗口丢弃）** | 符合「第一次有效」语义 | 需配置窗口时长 | ✅ |
| LaunchMode.singleTop | 栈顶去重 | 只覆盖栈顶相同路由，不防连点不同路由 | 互补，非替代 |

**决策**：Throttle 首触发模式，配置项 `navigationThrottleMs`，默认 `0`（不节流，零行为变更）。

---

## 三、语义定义

```
navigationThrottleMs = 300

t=0ms    push('/detail')  → ✅ 立即执行，记录 lastExecutedAt
t=50ms   push('/settings') → ❌ 丢弃（窗口内）
t=100ms  pop()             → ❌ 丢弃
t=301ms  push('/settings') → ✅ 执行
```

| 属性 | 值 |
| --- | --- |
| 模式 | Leading-edge throttle（首触发） |
| 作用范围 | 所有 `FastNavigator` 静态 API |
| 默认值 | `0`（禁用） |
| 配置入口 | `FastRouterConfig(navigationThrottleMs: 300)` |

## 四、NavigationThrottle 伪代码

```dart
// internal/navigation_throttle.dart — 设计伪代码
class NavigationThrottle {
  NavigationThrottle({this.durationMs = 0});

  final int durationMs;
  DateTime? _lastExecutedAt;

  bool get isEnabled => durationMs > 0;

  /// 尝试获取执行权
  /// true  → 允许执行
  /// false → 被节流拦截，调用方直接 return
  bool tryAcquire() {
    if (!isEnabled) return true;

    final now = DateTime.now();
    final last = _lastExecutedAt;
    if (last != null && now.difference(last).inMilliseconds < durationMs) {
      return false;
    }
    _lastExecutedAt = now;
    return true;
  }
}
```

**刻意保持极简**：

- 不依赖 `Timer` / `Stream` — 无生命周期管理负担
- 不区分 API 类型 — push 和 pop 共享同一窗口（防止 push 后立即 pop 的抖动场景）
- 放在 `internal/` — 不对业务 export

## 五、FastNavigator 接入伪代码

```dart
// api/fast_navigator.dart — 节流接入
class FastNavigator {
  NavigationThrottle _throttle = NavigationThrottle();

  static void bind(
    FastRouterDelegate delegate,
    RouteRegistry registry, {
    int navigationThrottleMs = 0,
  }) {
    _instance._delegate = delegate;
    _instance._registry = registry;
    _instance._throttle = NavigationThrottle(durationMs: navigationThrottleMs);
  }

  bool _runIfAllowed(void Function() action) {
    if (!_throttle.tryAcquire()) return false;
    action();
    return true;
  }

  static void push(String location, {Object? extra, LaunchMode launchMode = LaunchMode.standard}) {
    _instance._checkInitialized();
    _instance._runIfAllowed(() {
      _instance._pushImpl(location, extra: extra, launchMode: launchMode);
    });
  }

  static void pop() {
    _instance._checkInitialized();
    _instance._runIfAllowed(() {
      _instance._delegate!.updateState(_instance._delegate!.state.pop());
    });
  }

  // replace / go / popUntil / pushNamed — 同样包裹 _runIfAllowed
}
```

**接入原则**：节流在**门面层**拦截，Domain 层 `NavigationState` 保持纯函数，不受影响——单测 State 时无需考虑节流。

## 六、FastRouterConfig 配置传递

```dart
// api/fast_router_config.dart
factory FastRouterConfig({
  required List<FastBaseRoute> routes,
  FastRoute? unknownRoute,
  String initialLocation = '/',
  PageTransition defaultTransition = const PlatformAdaptiveTransition(),
  AppType? appType,
  int navigationThrottleMs = 0,  // ← 新增
}) {
  final registry = RouteRegistry(routes: routes, unknownRoute: unknownRoute);
  final router = FastRouter(registry: registry, initialLocation: initialLocation, ...);

  FastNavigator.bind(
    router.delegate,
    router.registry,
    navigationThrottleMs: navigationThrottleMs,
  );

  return FastRouterConfig._(router.routerConfig);
}
```

## 八、架构师视角：为什么不在 Delegate 层做

| 层级 | 做节流的后果 |
| --- | --- |
| Delegate | Parser 深链、Middleware redirect 也被节流 — 误伤 |
| NavigationState | 纯函数层掺入时间副作用 — 单测污染 |
| **FastNavigator 门面** | 只拦截业务主动导航 — ✅ 精确 |

Middleware redirect（M3）走 Delegate 内部路径，**不经过** `FastNavigator._runIfAllowed`——这是刻意设计。
