# 基本业务能力与接入示例

> **阶段定位**：M1 核心交付——Domain 模型接上 Router 四件套，业务 App 能 `push/pop/go`。  
> **验证标准**：BarrelLock Android 三页跳转 + Web 地址栏同步 + 浏览器后退正常。

---

## 一、心路历程：门面 API 怎么设计

Navigator 2.0 的正确用法是变换 `NavigationState`，但业务开发者不应每次：

```dart
// ❌ 业务层不应写的代码
final match = registry.matchLocation(Uri.parse('/detail'));
delegate.updateState(delegate.state.push(match));
```

**决策**：`FastNavigator` 单例门面，无 Context 全局调用，内部三步：

```
registry.match → NavigationState 变换 → delegate.updateState
```

对内**从不**调用 `Navigator.push`。

## 二、FastNavigator API 设计伪代码

```dart
// api/fast_navigator.dart — M1 设计伪代码
class FastNavigator {
  static final FastNavigator _instance = FastNavigator._internal();
  factory FastNavigator() => _instance;

  FastRouterDelegate? _delegate;
  RouteRegistry? _registry;

  static void bind(FastRouterDelegate delegate, RouteRegistry registry) {
    _instance._delegate = delegate;
    _instance._registry = registry;
  }

  // ── Push ──

  static void push(String location, {Object? extra, LaunchMode launchMode = LaunchMode.standard}) {
    _instance._checkInitialized();
    _instance._pushImpl(location, extra: extra, launchMode: launchMode);
  }

  static void pushNamed(
    String name, {
    Map<String, String>? pathParams,
    Map<String, String>? queryParams,
    Object? extra,
    LaunchMode launchMode = LaunchMode.standard,
  }) {
    final route = _instance._registry!.findByName(name);
    var path = route.path;
    pathParams?.forEach((k, v) => path = path.replaceAll(':$k', v));
    final uri = Uri(path: path, queryParameters: queryParams);
    _instance._pushImpl(uri.toString(), extra: extra, launchMode: launchMode);
  }

  void _pushImpl(String location, {Object? extra, LaunchMode launchMode = LaunchMode.standard}) {
    final match = _registry!.matchLocation(Uri.parse(location), extra: extra);
    final newState = _delegate!.state.push(match, launchMode: launchMode);
    _delegate!.updateState(newState);
  }

  // ── Pop / Replace / Go / PopUntil ──

  static void pop() {
    _delegate!.updateState(_delegate!.state.pop());
  }

  static void replace(String location, {Object? extra}) {
    final match = _registry!.matchLocation(Uri.parse(location), extra: extra);
    _delegate!.updateState(_delegate!.state.replace(match));
  }

  static void go(String location, {Object? extra}) {
    final match = _registry!.matchLocation(Uri.parse(location), extra: extra);
    _delegate!.updateState(_delegate!.state.go([match]));
  }

  static void popUntil(bool Function(String routeName) predicate) {
    _delegate!.updateState(
      _delegate!.state.popUntil((m) => predicate(m.route.name)),
    );
  }
}
```

### API 行为表

| API | NavigationState 操作 | 典型场景 |
| --- | --- | --- |
| `push` | 栈顶追加 | 进入详情 |
| `pop` | 弹出栈顶 | 返回上一页 |
| `replace` | 替换栈顶 | 登录成功后替换为首页 |
| `go` | 清栈压入新页 | Tab 切换、深链直达 |
| `popUntil` | 弹至 predicate 为 true | 回到指定页面 |
| `pushNamed` | push + 路由名解析 | 业务常量引用 |

## 三、FastRouterConfig — 一键接入

```dart
// api/fast_router_config.dart — M1 设计伪代码
class FastRouterConfig {
  final RouterConfig<NavigationState> routerConfig;

  factory FastRouterConfig({
    required List<FastBaseRoute> routes,
    FastRoute? unknownRoute,
    String initialLocation = '/',
  }) {
    final registry = RouteRegistry(routes: routes, unknownRoute: unknownRoute);

    final router = FastRouter(
      registry: registry,
      initialLocation: initialLocation,
    );

    // 绑定全局门面
    FastNavigator.bind(router.delegate, router.registry);

    return FastRouterConfig._(router.routerConfig);
  }
}
```

### 4.2 AppRoutes + AppRouter 单例

```dart
// apps/BarrelLock/BarrelLock_android/lib/router/router.dart — 目标形态
abstract final class AppRoutes {
  static const home = 'home';
  static const homePath = '/';
  static const detail = 'detail';
  static const detailPath = '/detail';
  static const settings = 'settings';
  static const settingsPath = '/settings';
}

class AppRouter {
  AppRouter._();
  static final AppRouter instance = AppRouter._();

  static RouterConfig<Object> get routerConfig {
    instance._ensureInitialized();
    return instance._fastRouterConfig.routerConfig;
  }

  void _ensureInitialized() {
    if (_initialized) return;
    _fastRouterConfig = FastRouterConfig(
      initialLocation: AppRoutes.homePath,
      routes: [
        FastRoute(
          name: AppRoutes.home,
          path: AppRoutes.homePath,
          builder: (_, m) => const HomePage(),
        ),
        FastRoute(
          name: AppRoutes.detail,
          path: AppRoutes.detailPath,
          builder: (_, m) => const DetailPage(),
        ),
        FastRoute(
          name: AppRoutes.settings,
          path: AppRoutes.settingsPath,
          builder: (_, m) => const SettingsPage(),
        ),
      ],
      unknownRoute: FastRoute(
        name: 'unknown',
        path: '/404',
        builder: (_, m) => const UnknownRoutePage(),
      ),
    );
    _initialized = true;
  }

  // 导航门面 — 委托 FastNavigator
  static void push(String location) => FastNavigator.push(location);
  static void pop() => FastNavigator.pop();
  static void go(String location) => FastNavigator.go(location);
}
```

## 五、M1 Page 层（过渡前）

M1 阶段 Delegate 直接用单一 Page 类型：

```dart
// M1 — router/fast_page.dart（M2 已移除）
class FastPage extends Page {
  final RouteMatch match;

  @override
  Route createRoute(BuildContext context) {
    return MaterialPageRoute(
      settings: this,
      builder: (ctx) => match.route.builder(ctx, match),
    );
  }
}

// fast_router_delegate.dart — M1 build()
final pages = _state.matches.map((m) => FastPage(match: m)).toList();
return Navigator(pages: pages, onDidRemovePage: _onDidRemovePage);
```

**M1 的已知局限**（驱动 M2 动画改造）：

- 所有路由共用 Material 默认动画
- CupertinoApp 下动画与平台不一致
- 无法单路由禁用动画

## 六、404 兜底

```dart
// 注册 unknownRoute
FastRouterConfig(
  routes: [...],
  unknownRoute: FastRoute(
    name: 'unknown',
    path: '/404',
    builder: (_, m) => UnknownRoutePage(path: m.path),
  ),

// RouteRegistry.matchLocation — 匹配失败
if (unknownRoute != null) {
  return RouteMatch(
    route: unknownRoute!,
    path: location.path,
    key: 'unknown_route',
    ...
  );

```

**原则**：匹配失败 → 404 Page，**避免路由崩溃 throw**。

## 七、M1 实现过程中的关键调试点

### 7.1 手势返回后 State 不同步

**现象**：iOS 侧滑返回后，再 `push` 出现重复页面。

**原因**：`onDidRemovePage` 未同步 `_state`。

**修复伪代码**：

```dart
void _onDidRemovePage(Page page) {
  final removedKey = (page.key as ValueKey<String>?)?.value;
  if (removedKey != null) {
    final newMatches = _state.matches.where((m) => m.key != removedKey).toList();
    updateState(NavigationState(matches: List.unmodifiable(newMatches)));
  } else {
    updateState(_state.pop());  // fallback
  }
}
```

### 7.2 Web 刷新丢栈

**现象**：push 到 `/detail` 后刷新，回到 `/detail` 单页（中间页丢失）。

**结论**：Location 模式下 URL = 栈顶，刷新重建单页栈是**预期行为**。产品需接受或改用 Full Stack URL 编码（M4+）。

### 7.3 push 不更新 URL

**现象**：Web 上 push 后地址栏不变。

**检查**：`currentConfiguration` 是否正确返回新 State → `restoreRouteInformation` 是否被调用。
