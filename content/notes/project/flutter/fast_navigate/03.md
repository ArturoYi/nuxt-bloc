# 分层架构、目录结构与领域模型

> **阶段定位**：M1 编码第一天——先画分层、定目录、写领域模型伪代码，**不写 Widget**。  
> **架构原则**：依赖单向向下，Domain 层零 Flutter Material 依赖。

---

## 一、心路历程：为什么要分层

自研路由最大的诱惑是「一个 RouterDelegate 文件写完」。短期快，长期会在以下场景爆炸：

- 单测 NavigationState 却要 mock BuildContext
- 改 Page Key 策略却要改 Parser
- 加 Middleware 发现逻辑散落在 push 方法里

**决策**：严格四层，Domain 纯 Dart，Router 只组装 2.0 四件套，Facade 面向业务。

```
┌──────────────────────────────────────────────────────────────┐
│  Application Layer（业务 App）                                │
│  AppRoutes 常量 + AppRouter 单例 + 页面 Widget                │
└───────────────────────────┬──────────────────────────────────┘
                            │
┌───────────────────────────▼──────────────────────────────────┐
│  Facade Layer（api/）                                          │
│  FastNavigator          → push / pop / go / replace / ...     │
│  FastRouterConfig       → MaterialApp.router 一键接入          │
└───────────────────────────┬──────────────────────────────────┘
                            │
┌───────────────────────────▼──────────────────────────────────┐
│  Router Layer（router/）— Navigator 2.0 四件套                │
│  FastRouterDelegate     → 栈管理、build Navigator、popRoute    │
│  FastRouteInformationParser → URL ↔ NavigationState           │
│  FastRouter             → 组合 Delegate + Parser + Dispatcher │
└───────────────────────────┬──────────────────────────────────┘
                            │
┌───────────────────────────▼──────────────────────────────────┐
│  Domain Layer（domain/）— 路由领域模型                          │
│  NavigationState        → 不可变栈状态（SSOT）                   │
│  RouteMatch             → 单次匹配结果                           │
│  RouteRegistry          → 路由表 + 匹配引擎                      │
│  RouteMiddleware        → 拦截管道（M3）                         │
│  LaunchMode             → Push 去重策略                          │
└───────────────────────────┬──────────────────────────────────┘
                            │
┌───────────────────────────▼──────────────────────────────────┐
│  Page Layer（page/）— 渲染基础                                  │
│  PageFactory            → match → Page（M2）                    │
│  RouteParameters        → path / query / extra                  │
└──────────────────────────────────────────────────────────────┘
```

## 三、核心领域模型伪代码

### 3.1 RouteParameters — 三层参数体系

```
location: /user/42?tab=info
          ─────  ──  ──────
          path   │   queryParams
               pathParams(:id)

push('/order', extra: OrderDraft)  →  extra（内存绑定，不写 URL）
```

```dart
// domain/match/route_parameters.dart — 设计伪代码
@immutable
class RouteParameters {
  final Map<String, String> pathParams;
  final Map<String, String> queryParams;
  final Object? extra;  // 不可序列化，不参与 URL

  const RouteParameters({
    this.pathParams = const {},
    this.queryParams = const {},
    this.extra,
  });
}
```

| 参数类型 | URL | 刷新可恢复 |
| --- | --- | --- |
| 固定段 / 动态 `:id` | ✅ | ✅ |
| `?key=val` query | ✅ | ✅ |
| `extra` | ❌ | ❌ |

### 3.3 RouteMatch — 参数持久绑定单元

```dart
// domain/match/route_match.dart — 设计伪代码
@immutable
class RouteMatch {
  final FastRoute route;
  final String path;              // 实际路径，如 /user/42
  final RouteParameters parameters;
  final String key;               // Page Diff 用，extra 不进 key

  Uri get uri => Uri(path: path, queryParameters: parameters.queryParams);

  /// 逻辑身份（不含 uniqueId），用于 LaunchMode 去重判断
  String get identity => generateKey(route.name, path);

  static String generateKey(String name, String path, {String? uniqueId}) {
    return uniqueId != null ? '$name:$path[$uniqueId]' : '$name:$path';
  }
}
```

**Page Key 策略（设计阶段定死）**：

| 场景 | Key 规则 | 原因 |
| --- | --- | --- |
| 同路由不同 id | `routeName:path` | 不同实例，必须重建 |
| 同路由 push 两次 | 加 `uniqueId` / stackIndex | 防 Page 复用 |
| extra 变化 | **不纳入 Key** | 改 Key 会错误 dispose |

### 3.4 NavigationState — SSOT 唯一真相源

```dart
// domain/state/navigation_state.dart — M1 设计伪代码（核心栈操作）
@immutable
class NavigationState {
  final List<RouteMatch> matches;  // 从底到顶

  const NavigationState({required this.matches});

  Uri get location => matches.isEmpty
      ? Uri(path: '/')
      : matches.last.uri;

  RouteMatch? get topMatch => matches.isNotEmpty ? matches.last : null;

  // ── 纯函数式栈操作，均返回新实例 ──

  NavigationState push(RouteMatch match, {LaunchMode launchMode = LaunchMode.standard}) {
    // standard: 重复 identity → throw DuplicateRouteException
    // singleTop: 栈顶相同 → 更新参数；否则 push
    // singleTask: 存在 → 弹出上层复用；否则 push
    // multipleTop: 存在 → 加 uniqueId 再 push
    return NavigationState(matches: List.unmodifiable([...matches, match]));
  }

  NavigationState pop() {
    if (matches.isEmpty) return this;
    return NavigationState(matches: List.unmodifiable(matches.take(matches.length - 1)));
  }

  NavigationState replace(RouteMatch match) { /* 替换栈顶 */ }
  NavigationState go(List<RouteMatch> newMatches) { /* 清栈 */ }
  NavigationState popUntil(bool Function(RouteMatch) predicate) { /* 弹至条件 */ }
}
```

**编码顺序**：先写 `push/pop/replace/go` 四个 primitive，单测通过后再写 `LaunchMode` 四个变体。

### 3.5 LaunchMode — Push 去重策略

```dart
// domain/state/launch_mode.dart
enum LaunchMode {
  standard,     // 重复 → DuplicateRouteException
  singleTop,    // 栈顶相同 → 更新参数
  singleTask,   // 存在 → 弹出上层复用实例
  multipleTop,  // 存在 → 新实例（unique key）
}
```

对标 Android `Intent` flags，解决「详情页连点 push 两次」类产品问题。

### 3.6 RouteRegistry — 匹配引擎

```dart
// domain/registry/route_registry.dart — 设计伪代码
class RouteRegistry {
  final Map<String, FastBaseRoute> _routesByName = {};
  final List<FastRoute> _leafRoutes = [];  // 按 path 长度降序 — 最长前缀优先
  FastRoute? unknownRoute;

  void addRoute(FastBaseRoute route) {
    _routesByName[route.name] = route;
    if (route is FastRoute) {
      _leafRoutes.add(route);
      _leafRoutes.sort((a, b) => b.path.length.compareTo(a.path.length));
    }
  }

  FastBaseRoute? findByName(String name) => _routesByName[name];

  RouteMatch matchLocation(Uri location, {Object? extra}) {
    for (final route in _leafRoutes) {
      final (isMatch, pathParams) = _tryMatch(route.path, location.path);
      if (isMatch) {
        return RouteMatch(
          route: route,
          path: location.path,
          parameters: RouteParameters(
            pathParams: pathParams,
            queryParams: location.queryParameters,
            extra: extra,
          ),
          key: RouteMatch.generateKey(route.name, location.path),
        );
      }
    }
    // 404 兜底 — 永不 throw（若配置了 unknownRoute）
    return _unknownMatch(location, extra);
  }
}
```

**M1 刻意简化**：只做叶子路由精确段匹配，不做 Shell 嵌套——留 M4。

## 四、Router 层组装伪代码

```dart
// router/fast_router.dart — 设计伪代码
class FastRouter {
  late final FastRouteInformationParser parser;
  late final FastRouterDelegate delegate;
  final RouteRegistry registry;

  FastRouter({required this.registry, String initialLocation = '/'}) {
    parser = FastRouteInformationParser(registry: registry);

    final initialMatch = registry.matchLocation(Uri.parse(initialLocation));
    delegate = FastRouterDelegate(
      initialState: NavigationState(matches: [initialMatch]),
    );
  }

  RouterConfig<NavigationState> get routerConfig => RouterConfig(
    routerDelegate: delegate,
    routeInformationParser: parser,
    routeInformationProvider: PlatformRouteInformationProvider(
      initialRouteInformation: RouteInformation(uri: delegate.state.location),
    ),
    backButtonDispatcher: RootBackButtonDispatcher(),
  );
}
```
